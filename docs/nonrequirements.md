# 非機能要件

## 1. 実行基盤
- クラウド基盤は Azure を利用する。
- デプロイ先リージョンは東日本リージョンとする。
- 実行環境は Azure Functions の Consumption Plan とする。
- 開発言語は Python 3.11 とする。

## 2. 可用性
- Azure Functions Consumption Plan の特性により、アイドル後の初回呼び出し遅延（コールドスタート）が発生し得ることを許容する。
- 単一リージョン（東日本）で運用する。

## 3. 性能
- 軽量な四則演算 API として、通常利用時に短時間で応答できること。
- 想定外の高負荷時は、プラットフォーム制限により応答遅延が発生する可能性を許容する。

## 4. 運用・監視
- 実行ログ（リクエスト、結果、エラー）を取得できること。
- エラー発生時に原因調査可能な最小限の情報をログに残すこと。
- 監視基盤（例: Application Insights）で稼働状況を確認可能とすること。

## 5. セキュリティ
- 入力値 A, B のバリデーションを行い、不正入力を受け付けないこと。
- エラーレスポンスには内部実装の詳細情報を含めないこと。
- 必要に応じて Azure Functions の認証レベル（Function/Anonymous 等）を設計時に決定すること。

## 6. 保守性
- API の責務（掛け算/割り算）を明確に分離し、変更影響範囲を限定できる構成とすること。
- 将来的な機能追加（加算・減算など）を想定し、拡張しやすい命名・構成を採用すること。
